# FlowPilot

## 项目简介
FlowPilot 是一款面向 AI 新手型职场者的 C 端 PC Web AI 工作流工具。它以高频职场任务为入口，通过上下文构建、结构化 Prompt、输出质检、版本优化、结果沉淀和 Skill Records，帮助用户把分散、随机、不可复用的 AI 使用过程，转化为清晰、稳定、可复用的个人工作方法。

与以自由对话为中心的 AI 产品不同，FlowPilot 并不将“输入一句话并等待结果”视为主要交互模型，而是围绕真实任务场景组织完整流程。用户从具体任务出发，在系统引导下逐步补全背景、目标、对象、素材和限制条件，形成结构化输入，再对生成结果进行检查、修订、分版本输出和方法留存，从而建立更可靠的工作路径。

## 核心用户
- 刚开始将 AI 用于日常工作的职场人
- 缺少稳定提问结构和结果判断标准的用户
- 需要频繁处理汇报、复盘、策划、沟通、整理类任务的人

## 核心问题
- 用户知道 AI 能做事，但难以从真实任务快速起步
- 用户缺少系统化的上下文补全能力，导致结果泛化、失焦或不贴合场景
- 用户拿到首轮输出后，缺少明确的质检标准与后续优化路径
- 用户即使偶尔获得较好结果，也难以沉淀为可持续复用的方法资产

## 核心流程
1. 用户从高频职场任务入口进入具体工作流。
2. 系统引导用户补全任务背景、目标、对象、素材与限制条件。
3. 系统将输入组织为可执行的结构化 Prompt。
4. 用户基于生成结果进行输出质检，识别缺漏、模糊点与风险。
5. 用户进一步完成版本优化，生成适配不同场景的输出版本。
6. 结果、方法和关键操作记录被沉淀到结果库与 Skill Records 中，供后续复用。

## 产品约束
- FlowPilot 不是课程 App，不以教学内容和学习进度为核心。
- FlowPilot 不是普通聊天工具，不以自由对话作为唯一入口。
- FlowPilot 不是游戏化成长产品，不使用等级化激励机制。
- Skill 保留为方法记录体系，用于标签、提示、记录与复盘，不用于状态分层。
- 工作台必须保留左侧可折叠导航、气球任务入口、绑线、聊天入口和小剪刀交互。
- 当前信息结构以本地数据驱动为主，不依赖后端即可完成核心流程验证。
